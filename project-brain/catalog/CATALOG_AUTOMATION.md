# RAG Catalog Automation & Vector Retrieval

This document outlines the ingestion, vectorization, and semantic retrieval architecture for business catalogs, FAQs, menus, and document stores.

---

## 1. Document Ingestion Pipeline

The platform supports unstructured and structured document formats: **PDF, DOCX, TXT, JSON, and CSV**.

The file processing flow is fully decoupled:
```
 [User Upload (/knowledge/{kb_id}/documents)]
                    │
                    ▼
[Store File on Disk (/app/uploads)] ──> [Save "processing" Doc State]
                    │
                    ▼ (Trigger Celery Vectorization Task)
 [process_kb_document_task (Celery Worker)]
                    │
                    ▼
[Text Extraction (PdfReader / File Parsers)]
                    │
                    ▼
[Overlapping Text Chunking (~500 chars with ~50 overlap)]
                    │
                    ▼
[Generate Vectors (Ollama: all-minilm embeddings)]
                    │
                    ▼
 [PostgreSQL pgvector Storage (kb_document_chunks)]
                    │
                    ▼
      [Status Updated to "processed"]
```

---

## 2. Vector Search & Similarity Retrieval

* **Database Engine**: PostgreSQL with `pgvector` extension.
* **Vector Dimension**: `384` dimensions generated by `all-minilm:latest` embedding model.
* **Retrieval Query**: Cosine distance operator (`<=>`) matches query vector against chunks linked to the tenant's knowledge store:
  ```sql
  SELECT chunk.content, chunk.embedding <=> :query_vector AS distance
  FROM kb_document_chunks chunk
  JOIN kb_documents doc ON chunk.document_id = doc.id
  WHERE doc.kb_id = :kb_id
  ORDER BY distance ASC
  LIMIT :limit
  ```

## 3. Product Catalog & Pricing Automation

When a customer inquires about catalog pricing or products (e.g. *"I want ₹50 pakoda"*):
1. **Semantic Hit**: Similarity search resolvespakoda prices and variations from catalog files.
2. **Context Assembly**: Chunk content is injected into LLM instructions.
3. **Conversational Synthesis**: LLM parses the numerical pricing details, calculates amounts, suggests logical add-ons, and responds in a direct, human-like sales conversational tone.
